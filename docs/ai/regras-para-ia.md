# Regras para IA

## Leitura obrigatória antes de alterar código

Antes de alterar código, ler:

- `docs/README.md`
- `docs/00-LEIA-PRIMEIRO.md`
- `docs/project-status.md`
- `docs/ai/contexto-do-projecto.md`

## Regras obrigatórias

- nunca inventar funcionalidades;
- nunca dizer que algo existe se for apenas planeado;
- preservar português como língua principal da UX;
- preservar abordagem mobile-first;
- preservar distinção entre cidadão e admin;
- actualizar documentação sempre que alterar código;
- actualizar `changelog.md` sempre que alterar documentação ou comportamento;
- não remover funcionalidades existentes sem autorização explícita.
